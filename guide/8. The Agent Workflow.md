# The Agent Workflow

DXEditor is built to be the human end of an agentic content workflow with
Claude Code. If you are not using a pipeline yet, you can skip this page.

## How it works

1. Your pipeline writes drafts into a shared folder.
2. Reviewers read them here and leave comments. "For agent" comments are
   instructions for the pipeline. "For team" comments are for colleagues.
3. In the workspace menu, click **Add agent instructions** once per folder.
   This writes an `AGENTS.md` that tells any Claude Code run how to find open
   agent comments, apply them, mark them resolved, and reply with what
   changed.
4. When the agent resolves a comment, your screen updates live: the draft text
   changes, the highlight clears, and the comment shows the agent's reply.

## Why it is useful

Resolved comments build up a feedback history the pipeline can mine for
recurring guidance on tone, length, and structure, and apply it proactively
next time.
